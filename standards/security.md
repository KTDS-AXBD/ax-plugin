# 보안 표준

> 범용 표준 — 모든 프로젝트에 공통 적용

## 1. 시크릿/환경변수 관리

### 환경별 분리

| 환경 | 저장소 | 접근 |
|------|---------|------|
| 로컬 | `.env` / `.dev.vars` (gitignored) | 개발자만 |
| 프로덕션 | 플랫폼 시크릿 관리자 (wrangler secret, Vercel env 등) | 배포 시스템 |
| CI/CD | CI 환경변수 (GitHub Actions secrets 등) | 파이프라인 |

### 금지 사항

- 시크릿을 코드에 하드코딩 금지
- `.env` 파일을 git에 커밋 금지
- 로그/에러 메시지에 시크릿 포함 금지
- 시크릿을 URL 쿼리 파라미터로 전달 금지

### 접근 패턴

- 환경변수 접근은 래퍼 함수 경유 권장 (직접 `process.env` 접근 최소화)
- `.env.example` 파일로 필요한 환경변수 목록 공유 (값은 제외)
- 시크릿 로테이션 주기 정의 권장

## 2. 인증/인가

### 가드 계층

| 레벨 | 설명 | 미충족 시 처리 |
|------|------|----------------|
| public | 인증 불필요 | 제한 없음 |
| user | 로그인 필요 | 로그인 페이지 리디렉트 |
| role | 특정 역할 필요 | 403 Forbidden |
| admin | 관리자 전용 | 403 Forbidden |

### 인증 규칙

- **가드 함수 필수**: 라우트/API에 인증 가드 명시적 적용. 무인증 접근 가능한 경로는 의도적으로 public 표시
- **기본값은 인증 필요**: 새 라우트는 기본적으로 인증 필요. public은 의도적 선택
- **가드 함수 명명**: `requireUser()`, `requireAdmin()` 등 명확한 이름

### 세션 규칙

- 세션 토큰: httpOnly + secure + sameSite 속성 필수
- 세션 만료 정책 정의 필수 (프로젝트별 기간 결정)
- 세션 데이터에 민감 정보(비밀번호, 카드번호 등) 저장 금지
- 세션 ID는 충분한 엔트로피 보장 (최소 128bit)

## 3. 입력 검증

### 경계 검증 원칙

- **API 경계에서 검증**: 모든 외부 입력은 API 진입점에서 검증
- **스키마 기반 검증**: zod, yup 등 스키마 라이브러리로 구조적 검증
- **내부 코드 신뢰**: 검증된 후 내부에서 재검증 불필요

### OWASP Top 10 대응

| 위협 | 대응 |
|------|------|
| XSS | 사용자 입력 출력 시 이스케이핑. 프레임워크 기본 보호 활용, raw HTML 삽입 금지 |
| SQL Injection | ORM/쿼리 빌더 사용 필수. raw SQL에 사용자 입력 직접 삽입 금지 |
| CSRF | SameSite 쿠키 + CSRF 토큰 (필요 시) |
| 인증 우회 | 가드 함수 경유 필수. 직접 DB 조회로 권한 체크 금지 |
| 민감 데이터 노출 | API 응답에 불필요한 필드 제외. 비밀번호 해시 등 절대 반환 금지 |
| 보안 설정 오류 | 프로덕션에서 디버그 모드 비활성화. 에러 스택트레이스 노출 금지 |

### 파일 업로드

- 파일 타입 화이트리스트 검증
- 파일 크기 제한 설정
- 업로드된 파일은 별도 스토리지에 저장 (앱 디렉토리 외부)

## 4. 데이터 보호

### 민감 정보 분류

| 등급 | 예시 | 처리 |
|------|------|------|
| 극비 | 비밀번호, API 키, 토큰 | 암호화 저장, 로그 금지, 평문 전송 금지 |
| 민감 | 이메일, 전화번호, 이름 | 접근 제어, 로그 시 마스킹 |
| 일반 | 공개 게시글, 설정값 | 일반 보호 |

### 로깅 시 마스킹

- 시크릿/토큰: 완전 마스킹 (`****`)
- 이메일: 부분 마스킹 (`s***@gmail.com`)
- 개인정보: 부분 마스킹 또는 해싱
- 에러 로그에 request body 전체 포함 금지

### 암호화

- 비밀번호: bcrypt/argon2 등 단방향 해싱 (평문 저장 금지)
- 민감 데이터 전송: HTTPS 필수
- DB 내 민감 데이터: 필요 시 컬럼 레벨 암호화

## 5. 자동화

### Claude Code 훅

- `.env` / 시크릿 파일 커밋 방지 훅
- 하드코딩된 시크릿 패턴 감지 (API 키 형태, password= 등)

### 권장 실천

- `npm audit` / `pnpm audit` 정기 실행 (의존성 취약점 점검)
- `.gitignore`에 시크릿 파일 패턴 포함 확인
- 보안 관련 의존성 업데이트 우선 처리

## 6. AI 협업

### 시크릿 노출 금지

- AI에게 실제 시크릿 값을 직접 전달하지 않음
- 환경변수 참조(`process.env.API_KEY`, `context.cloudflare.env.KEY`)만 사용
- 코드에 시크릿이 하드코딩되어 있으면 즉시 환경변수로 전환

### 보안 코드 리뷰

- AI가 코드 작성 시 OWASP 취약점 자체 점검
- 사용자 입력을 다루는 코드에서 검증/이스케이핑 확인
- 인증 가드 누락 감지 시 경고
