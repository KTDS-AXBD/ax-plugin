# 온보딩/지식 공유 표준

> 범용 표준 — 모든 프로젝트에 공통 적용

## 1. 새 프로젝트 셋업

### 필수 파일 체크리스트

| 파일 | 용도 | 최초 작성 시점 |
|------|------|---------------|
| `CLAUDE.md` | 프로젝트 규칙, 명령어, Gotchas | 프로젝트 시작 시 |
| `SPEC.md` | 프로젝트 사양, 요구사항 | 프로젝트 정의 시 |
| `MEMORY.md` | 활성 컨텍스트, 작업 추적 | 첫 세션부터 자동 |
| `package.json` | 의존성, 스크립트, 버전 | 프로젝트 시작 시 |
| `.gitignore` | Git 제외 파일 | 프로젝트 시작 시 |
| `.env.example` | 필요한 환경변수 목록 (값 제외) | 환경변수 추가 시 |
| `docs/CHANGELOG.md` | 세션별 변경 이력 | 첫 세션부터 |
| `docs/INDEX.md` | 문서 인덱스 | 문서 추가 시 |

### 거버넌스 표준 적용

새 프로젝트 시작 시 범용 표준은 자동 적용 (`~/.claude/CLAUDE.md` 경유). 프로젝트별 확장 사항만 프로젝트 `CLAUDE.md`에 추가:

```
~/.claude/CLAUDE.md          ← 범용 표준 (자동)
  └─ {project}/CLAUDE.md     ← 프로젝트 확장 (수동)
```

### 개발 환경 셋업

1. 리포지토리 클론
2. 패키지 매니저 설치 (package.json의 `packageManager` 참조)
3. 의존성 설치 (`pnpm install` 등)
4. 환경변수 설정 (`.env.example` → `.env` 복사 후 값 채우기)
5. 로컬 DB 셋업 (필요 시)
6. 개발 서버 실행 확인 (`dev` 스크립트)
7. 테스트 실행 확인 (`test` 스크립트)

### 초기 커밋 가이드

```
1. 프로젝트 초기화 (프레임워크 scaffolding)
2. CLAUDE.md 작성 (프로젝트 규칙, 명령어, 스택)
3. .gitignore 설정
4. 첫 커밋: `chore: initialize project`
5. SPEC.md 작성 (프로젝트 사양)
6. 두 번째 커밋: `docs: add SPEC.md`
```

## 2. 새 팀원 합류

### 합류 체크리스트

1. **읽기** — CLAUDE.md → SPEC.md → MEMORY.md 순서로 읽기
2. **환경** — 개발 환경 셋업 (위 §1 절차)
3. **실행** — `dev`, `test`, `build` 명령어 실행 확인
4. **이해** — 디렉토리 구조, 핵심 파일 파악
5. **기여** — 작은 이슈부터 시작하여 첫 커밋

### 핵심 파악 순서

```
CLAUDE.md (규칙/가이드)
  → SPEC.md (무엇을 만드는가)
    → 디렉토리 구조 (어떻게 구성되어 있는가)
      → 핵심 비즈니스 로직 (어떻게 동작하는가)
```

## 3. AI Agent 온보딩

### 자동 컨텍스트 로딩

AI Agent(Claude Code 등)는 세션 시작 시 자동으로 컨텍스트를 파악:

- `CLAUDE.md` — 프로젝트 규칙, 금지 사항, 패턴
- `MEMORY.md` — 현재 상태, 최근 작업, 다음 작업

### AI가 빠르게 파악하는 구조

| 정보 | 위치 | AI 접근 방식 |
|------|------|-------------|
| 프로젝트 규칙 | CLAUDE.md | 자동 로딩 |
| 현재 컨텍스트 | MEMORY.md | 자동 로딩 |
| 프로젝트 사양 | SPEC.md | 세션 시작 시 읽기 |
| 기술 결정 이유 | ADR 문서 | 필요 시 검색 |
| 변경 이력 | CHANGELOG.md | 필요 시 검색 |
| 구현 상세 | 코드 자체 | 작업 시 읽기 |

### CLAUDE.md 작성 가이드 (AI용)

- **명령어 섹션**: 빌드/테스트/배포 명령어 목록
- **Gotchas 섹션**: 실수하기 쉬운 패턴과 올바른 패턴
- **금지 사항**: 절대 하면 안 되는 것 명시
- **디렉토리 구조**: 어디에 무엇이 있는지 개요

## 4. 지식 공유 채널 (3-Tier)

### Tier 1: 자동 로딩 (세션 시작 시)

| 파일 | 역할 | 갱신 주기 |
|------|------|-----------|
| CLAUDE.md | 프로젝트 규칙, 가이드, Gotchas | 규칙 변경 시 |
| MEMORY.md | 활성 컨텍스트, 작업 추적, 결정사항 | 매 세션 |

### Tier 2: 보충 읽기 (세션 시작 시 또는 필요 시)

| 파일 | 역할 | 갱신 주기 |
|------|------|-----------|
| SPEC.md | 프로젝트 사양, 요구사항, 지표 | 마일스톤 단위 |
| docs/ 문서 | 설계, 분석, 가이드, ADR | 작성 시 |

### Tier 3: 검색 시에만

| 파일 | 역할 | 갱신 주기 |
|------|------|-----------|
| CHANGELOG.md | 세션별 변경 이력 | 매 세션 |
| 코드 자체 | 구현 상세, 패턴, 테스트 | 상시 |

### 정보 흐름

```
세션 중 발견/결정
  → MEMORY.md (활성 컨텍스트)
    → CLAUDE.md (규칙으로 승격 시)
    → SPEC.md (사양 변경 시)
    → ADR (중대 결정 시)
  → CHANGELOG.md (세션 이력)
```
