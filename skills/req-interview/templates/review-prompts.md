# 외부 AI 검토 프롬프트 패키지

> requirements-interview 스킬이 각 AI 서비스에 보낼 검토 요청 프롬프트를 생성할 때 사용하는 템플릿.
> PRD 내용을 {PRD_CONTENT} 자리에 삽입하여 최종 프롬프트를 완성한다.

---

## AI별 역할 분담 원칙

| AI | 검토 관점 | 강점 |
|----|-----------|------|
| **ChatGPT** | 논리적 완결성, 실행 가능성 | 구조적 사고, 실무 검증 |
| **Gemini** | 시장/트렌드 관점, 유사 사례 | 검색 기반 사실 확인 |
| **DeepSeek** | 기술적 실현 가능성, 아키텍처 | 기술 리스크 분석 |

> Claude/Grok은 수동 검토용으로만 사용 (models.json 미등록). 필요 시 아래 프롬프트 참조.

## ChatGPT 프롬프트

```
당신은 IT 서비스 기획과 프로젝트 관리 경험이 풍부한 시니어 PM입니다.
아래 PRD를 검토하고 실무적 관점에서 평가해주세요.

---
{PRD_CONTENT}
---

다음 항목별로 검토 의견을 주세요:

1. **논리적 완결성**: 문제-해결책-성공기준의 논리 흐름이 일관된가요?
   - 결함이 있다면 구체적으로 어떤 부분인지

2. **실행 가능성**: 제시된 범위와 일정/인력이 현실적인가요?
   - 과소평가된 부분이 있다면?

3. **누락된 핵심 요소**: 이런 종류의 프로젝트에서 반드시 있어야 하는데 빠진 것은?

4. **리스크**: 가장 크게 우려되는 실패 요인은?

5. **착수 판단**: Ready / Not Ready / Conditional 중 하나로 판단하고 이유를 설명해주세요.
   - Conditional이라면 어떤 조건이 충족되어야 하는지 명시해주세요.

답변은 항목별로 구체적이고 실용적으로 작성해주세요. 칭찬보다 개선 포인트에 집중해주세요.
```

```
당신은 IT 시장 트렌드와 유사 서비스 분석을 전문으로 하는 애널리스트입니다.
아래 PRD를 검토하고 시장/트렌드/사례 관점에서 평가해주세요.

---
{PRD_CONTENT}
---

다음 항목별로 검토 의견을 주세요:

1. **시장 적합성**: 이 솔루션이 현재 시장/트렌드와 맞닿아 있나요?
   - 유사한 서비스나 제품이 있다면 어떤 차별점이 필요한가요?

2. **사실 확인**: PRD에 포함된 가정이나 수치 중 검증이 필요한 것들은?

3. **사용자 관점**: 목표 사용자가 실제로 이 솔루션을 받아들일 것 같나요?
   - 도입 장벽이 있다면?

4. **누락된 시장 관점**: 시장/경쟁/트렌드 측면에서 고려되지 않은 것은?

5. **착수 판단**: Ready / Not Ready / Conditional 중 하나로 판단하고 이유를 설명해주세요.

검색 가능한 최신 정보와 사례를 참조하여 답변해주세요.
```

```
당신은 요구사항 분석과 시스템 설계 전문가입니다.
아래 PRD를 검토하고 구조적 일관성과 숨겨진 가정을 중심으로 평가해주세요.

---
{PRD_CONTENT}
---

다음 항목별로 검토 의견을 주세요:

1. **구조적 일관성**: 문서 전체에서 서로 모순되거나 충돌하는 내용이 있나요?
   - 용어가 일관되게 사용되고 있나요?

2. **숨겨진 가정**: 명시되지 않았지만 성공을 위해 사실이어야 하는 가정들은?
   - 이 가정들이 틀릴 경우 어떻게 되나요?

3. **엣지 케이스**: 다루지 않은 예외 상황이나 경계 조건은?

   - "적절한", "빠른", "쉬운" 같은 주관적 표현이 있다면 명확화 필요

5. **착수 판단**: Ready / Not Ready / Conditional 중 하나로 판단하고 이유를 설명해주세요.

분석적이고 비판적으로 답해주세요. 문서를 있는 그대로 읽지 말고 가정에 도전해주세요.
```

---
```
{PRD_CONTENT}
---

다음 항목별로 검토 의견을 주세요:

1. **가장 큰 문제점**: 이 PRD에서 당신이 가장 마음에 걸리는 것은 무엇인가요?
   - 왜 이게 실패할 것 같은지

2. **과대평가된 부분**: 실제보다 쉽게 생각하고 있는 부분은?
   - 현실에서는 왜 더 어려운가?

3. **과소평가된 복잡성**: 겉보기보다 훨씬 복잡한 부분은?

4. **이해관계자 문제**: 사람들이 실제로 이걸 사용할까요? 저항이 생길 수 있는 곳은?

5. **착수 판단**: Ready / Not Ready / Conditional 중 하나로 판단하고 이유를 설명해주세요.
   - 진행하려면 무조건 먼저 해결해야 할 것 하나만 꼽는다면?

필터 없이 솔직하게 말해주세요. 기분 좋은 피드백은 필요 없습니다.
```

## 검토 패키지 생성 규칙

Claude Code에서 이 파일을 참조할 때:

1. `{PRD_CONTENT}` 자리에 현재 PRD 전체 내용을 삽입한다
2. 각 AI별 파일을 별도로 생성한다 (`chatgpt-prompt.md`, `gemini-prompt.md` 등)
3. 하나의 통합본 `review-package-r{N}.md`도 생성한다 (전체 맥락 보존용)
4. 사용자에게 다음을 안내한다:
   - 각 파일의 내용을 해당 AI 서비스에 붙여넣기
   - 결과를 `feedback.md`에 저장하거나 직접 붙여넣기
   - Claude가 종합 분석 후 PRD 업데이트
