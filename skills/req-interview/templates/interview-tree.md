# 인터뷰 질문 트리

> Claude Code에서 requirements-interview 스킬이 사용하는 표준 인터뷰 가이드.
> 각 파트의 핵심 질문을 먼저 던지고, 답변에 따라 후속 질문을 선택적으로 진행한다.

---

## 인터뷰 시작 선언

```
안녕하세요. 요구사항 인터뷰를 시작할게요.
편하게 말씀해주세요. 음성으로 하셔도 됩니다.
먼저 이 프로젝트를 부를 이름이나 코드네임이 있나요?
```

---

## Part 1: 왜 (목적/문제)

**핵심 질문**
> "지금 어떤 문제가 있어서, 또는 어떤 기회를 잡으려고 이걸 만들려고 하시나요?"

**후속 질문 풀 (답변 내용에 따라 1~2개 선택)**
- "이 문제가 지금 어느 정도로 심각한가요? 측정 가능한 수치가 있다면 더 좋아요."
- "이 문제를 지금까지 해결하지 못한 이유가 뭔가요?"
- "이걸 만들지 않으면 어떻게 되나요?"
- "이 기회를 지금 잡아야 하는 이유가 있나요? (시급성)"

**파악해야 할 것**
- [ ] 해결하려는 핵심 문제 or 잡으려는 기회
- [ ] 현재 상태 (As-Is)
- [ ] 목표 상태 (To-Be)
- [ ] 시급성 또는 타이밍

---

## Part 2: 누구를 위해 (사용자/이해관계자)

**핵심 질문**
> "이걸 누가 사용하거나 영향을 받게 되나요?"

**후속 질문 풀**
- "주 사용자와 부 사용자를 구분할 수 있나요?"
- "의사결정권자(구매자/승인자)는 사용자와 다른 사람인가요?"
- "이해관계자 중 반대하거나 저항할 수 있는 사람이 있나요?"
- "사용자의 기술 수준이나 환경은 어떤가요? (비개발자? 모바일? 사내망?)"

**파악해야 할 것**
- [ ] 주 사용자 페르소나
- [ ] 이해관계자 맵
- [ ] 사용 환경/컨텍스트

---

## Part 3: 무엇을 (범위/기능)

**핵심 질문**
> "핵심 기능을 딱 한 문장으로 말한다면 뭔가요? 그리고 꼭 있어야 할 것과 나중에 해도 되는 것을 구분해줄 수 있나요?"

**후속 질문 풀**
- "이번 범위에 절대 포함하지 않을 것들도 정해두셨나요? (Out-of-scope)"
- "비슷한 서비스나 레퍼런스가 있다면 어떤 부분을 참고하고 싶으신가요?"
- "데이터는 어디서 오고 어디로 가나요?"
- "기존 시스템이나 서비스와 연동이 필요한가요?"

**파악해야 할 것**
- [ ] 핵심 기능 목록 (Must Have)
- [ ] 2차 기능 목록 (Should Have)
- [ ] 명시적 Out-of-scope
- [ ] 외부 연동/의존성

---

## Part 4: 어떻게 판단할 것인가 (성공 기준)

**핵심 질문**
> "이게 성공했다는 걸 어떻게 알 수 있을까요? 숫자로 측정 가능한 기준이 있다면 이상적입니다."

**후속 질문 풀**
- "6개월 후 이게 성공이라고 말할 수 있으려면 어떤 상태여야 하나요?"
- "최소한 이 정도는 되어야 한다는 기준이 있나요? (MVP 기준)"
- "실패 조건도 정의할 수 있나요? 언제 중단을 결정하시겠어요?"
- "성능/속도/안정성 같은 비기능적 요구사항이 있나요?"

**파악해야 할 것**
- [ ] 정량적 성공 지표 (KPI)
- [ ] MVP 최소 기준
- [ ] 실패/중단 조건
- [ ] 비기능 요구사항

---

## Part 5: 제약과 리소스 (현실 조건)

**핵심 질문**
> "시간, 예산, 인력, 기술 스택 등에서 알아야 할 제약 조건이 있나요?"

**후속 질문 풀**
- "이 프로젝트에 투입 가능한 인원과 기간은 어느 정도인가요?"
- "사용해야 하는 기술 스택이나 인프라 조건이 있나요?"
- "보안/규정/컴플라이언스 이슈가 있나요? (KT DS 내부 정책 포함)"
- "예산 규모 또는 비용 민감도는 어느 정도인가요?"
- "외부 승인이 필요한 과정이 있나요?"

**파악해야 할 것**
- [ ] 일정 제약
- [ ] 기술 제약
- [ ] 예산/인력 규모
- [ ] 컴플라이언스/보안 조건
- [ ] 의사결정/승인 프로세스

---

## 인터뷰 마무리

모든 파트 완료 후:
```
인터뷰가 완료되었습니다. 정리 내용을 확인해 주세요.

[요약 표시]

수정하거나 추가할 내용이 있으신가요?
없으시면 PRD 작성을 시작하겠습니다.
```

---

## 인터뷰 팁 (Claude가 준수할 규칙)

1. **한 번에 하나만**: 절대 여러 질문을 한꺼번에 던지지 않는다
2. **음성 친화적**: 구어체 답변도 그대로 받고 나중에 정리한다
3. **모른다고 막히지 않기**: "지금 모르셔도 됩니다, 나중에 추가할 수 있어요"
4. **예시 제공**: 답이 막히면 예시를 들어준다
5. **25분 제한 의식**: 인터뷰가 너무 길어지면 나머지는 PRD 작성 후 보완 제안
