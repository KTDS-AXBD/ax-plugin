# 착수 충분도 스코어카드 v3

> requirements-interview 스킬이 매 검토 라운드 종료 후 착수 준비 여부를 판단할 때 사용하는 기준.
> v2: 이슈 파싱 구조화, 비율 연속 스케일링, 3단계 충실도 평가 적용.
> v3: severity 가중치(flaw×3 + gap×1 + risk×1) + 라운드 간 품질 비교 + severity breakdown 표시.

---

## 스코어카드 항목

### 항목 1: 가중 이슈 밀도 (20점)

이전 라운드 대비 **가중 이슈 밀도**(가중 이슈 수 / PRD 1000자)의 변화를 기준으로 평가한다.

#### severity 가중치

이슈 유형별 심각도가 다르므로, 가중치를 부여하여 이슈 수를 환산한다.

| 유형 | 가중치 | 근거 |
|------|:------:|------|
| flaw (결함) | ×3 | 구현 시 반드시 수정 필요. 방치하면 사용자 영향 |
| gap (누락) | ×1 | 보완 필요하나 즉시 장애로 이어지지 않음 |
| risk (리스크) | ×1 | 인지/모니터링 대상. 발현 시 대응 필요 |

**가중 이슈 수 계산:** `weighted = flaw × 3 + gap × 1 + risk × 1`
**가중 밀도 계산:** `density = weighted / PRD길이(자) × 1000`

#### 밀도 변화 판정

| 가중 밀도 Δ (현재 - 이전) | 점수 |
|--------------------------|------|
| ≤ 0 (감소 또는 동일) | 20점 |
| 0 < Δ ≤ 1 (소폭 증가) | 14점 |
| 1 < Δ ≤ 2 (중간 증가) | 8점 |
| Δ > 2 (대폭 증가) | 0점 |

> **초안(Round 1)은 이 항목을 건너뛴다.** 처음엔 당연히 이슈가 많다.

#### 라운드 간 품질 비교

Round 2+에서는 이전 라운드의 severity 분포(`scorecard.json`)와 비교하여 **유형별 증감**을 표시한다.
예: `[flaw-2, gap+1, risk-1]` — flaw가 2건 줄고 gap이 1건 늘었음을 의미.

**v2 파싱 개선:** AI 응답에서 구체적 지적 항목을 추출할 때 번호 항목(`1.`, `2.`) 및 볼드 불릿(`- **항목:**`) 패턴을 인식하여 개별 카운트. 섹션 키워드만으로 분류하지 않고 항목 단위로 정밀 집계.

### 항목 2: Ready 판정 비율 (30점)

AI 참여 수에 관계없이 비율로 환산하여 0~30점 연속 스케일링.

| AI 판정 | 비율값 |
|---------|-------|
| Ready | 1.0 |
| Conditional | 0.5 |
| Not Ready | 0.0 |
| Unknown (파싱 실패) | 0.0 (참여 수에서 제외) |

**계산:** `readyPercent = sum(비율값) / parsedCount` → `score = round(readyPercent × 30)`

| 예시 | readyPercent | 점수 |
|------|-------------|------|
| 2개 모두 Ready | 100% | 30 |
| 2개 모두 Conditional | 50% | 15 |
| 1 Ready + 1 Conditional | 75% | 23 |
| 1 Ready + 1 Not Ready | 50% | 15 |
| 4개 모두 Ready | 100% | 30 |

> v1과 달리 AI 수에 따른 불이익 없음. 2개 AI 모두 Ready면 30점.

---

### 항목 3: 핵심 요소 커버리지 (30점)

PRD의 필수 항목을 **3단계 충실도**로 평가한다.

| 등급 | 조건 | 점수 |
|------|------|------|
| 충실 | 섹션 존재 + 품질 키워드 3개+ | 만점 |
| 최소 | 섹션 존재 + 품질 키워드 1~2개 | 절반 (올림) |
| 없음 | 섹션 미존재 또는 내용 부족 | 0점 |

| 필수 항목 | 만점 | 품질 키워드 |
|-----------|------|-----------|
| 핵심 문제 정의 | 5점 | as-is, to-be, 시급, 현재, 목표 |
| 주 사용자 및 이해관계자 | 5점 | 페르소나, 니즈, 역할, 사용 환경, 여정 |
| 핵심 기능 범위 | 5점 | p0, must, phase, 핵심 |
| 명시적 Out-of-scope | 4점 | 금지, 제외, 포함하지 |
| 정량적 성공 기준(KPI) | 6점 | 현재, 목표, 측정, 지표 |
| MVP 최소 기준 | 5점 | [ ], [x], 최소, 필수 |

---

### 항목 4: 다관점 반영 여부 (20점)

3개 관점의 **키워드 매칭 수**에 따라 충실도를 평가한다.

| 등급 | 키워드 매칭 수 | 점수 |
|------|-------------|------|
| 충실 | 3개 이상 | 만점 |
| 최소 | 1~2개 | 절반 (올림) |
| 없음 | 0개 | 0점 |

| 관점 | 만점 | 키워드 |
|------|------|--------|
| 사용자 관점 | 7점 | 사용자 경험, ux, 사용성, 니즈, 페르소나, 사용 환경, 시나리오, 여정, 온보딩, 접근성 |
| 기술 관점 | 7점 | 기술 스택, 아키텍처, api, 인프라, 프론트엔드, 백엔드, 데이터베이스, 기술적 제약, edge runtime, ssr |
| 비즈니스 관점 | 6점 | roi, 비용, 수익, 시장, 비즈니스, 예산, 경쟁, 차별화, 사업, 가치 |

## 종합 판정

| 총점 | 판정 | 권고 |
|------|------|------|
| 80점 이상 | ✅ **착수 준비 완료** | 최종 PRD 생성 후 착수 가능 |
| 60~79점 | 🔄 **추가 검토 권장** | 미달 항목 중심으로 1라운드 추가 |
| 59점 이하 | ⚠️ **검토 필수** | 핵심 항목 재정의 후 재검토 |

## 반복 종료 판단 가이드

**착수 가능 조건 (모두 충족 시):**
- [ ] 총점 80점 이상
- [ ] 항목 3 (커버리지) 24점 이상 (80% 이상)
- [ ] 연속 2라운드 총점 변화 ±5점 이내 (안정화)

**조기 종료 불가 조건 (하나라도 해당 시 계속):**
- Not Ready 판정 AI가 2개 이상
- 핵심 기능 범위 또는 성공 기준이 미확인 상태
- 신규 이슈가 계속 증가하는 추세

**최대 라운드:** 제한 없음. 단, 5라운드 이상 진행 시 근본적 재정의를 권고한다.
