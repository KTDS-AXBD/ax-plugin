# Ambiguity Score 기준서

> ouroboros 패턴 차용 — Foundry-X F59

## 공식

```
Ambiguity = 1 − Σ(clarityᵢ × weightᵢ)
```

- 값 범위: 0.0 (완벽히 명확) ~ 1.0 (완전히 모호)
- 각 차원별 clarity: 0.0 ~ 1.0

## 4차원 채점 기준

| 차원 | 설명 | 0.0 (모호) | 0.5 (보통) | 1.0 (명확) |
|------|------|-----------|-----------|-----------|
| **Goal** | 프로젝트 목표 | 목표 미정의 | 대략적 방향만 | 구체적 목표+측정 가능 |
| **Constraint** | 제약 조건 | 제약 미파악 | 일부 제약 인지 | 기술/비즈니스/시간 제약 명시 |
| **Success** | 성공 기준 | 기준 없음 | 정성적 기준만 | 정량적 KPI 정의됨 |
| **Context** | 기존 맥락 (brownfield) | 코드베이스 미파악 | 주요 모듈만 인지 | 아키텍처+의존성 완전 파악 |

## 가중치 표

## 판정 기준

| Ambiguity 범위 | 판정 | 조치 |
|---------------|------|------|
| ≤ 0.2 | **Ready** | 착수 가능, Plan 문서 작성 진행 |
| 0.2 ~ 0.4 | **주의** | 부족 차원 보완 질문 1~2회 추가 |
| > 0.4 | **미달** | Socratic 질문법으로 재인터뷰 필요 |

## 적용 위치

- `ax-14-req-interview` 스킬: 인터뷰 완료 후 자동 채점
- threshold ≤ 0.2 충족 시 PRD 작성 단계로 진행
- 미달 시 부족 차원을 명시하고 추가 질문 생성
